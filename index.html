Auditron

Proxy interceptor tipo Burp Suite · MITM addon para mitmproxy · Extracción automática de tokens · IA integrada (OpenAI)

Python 3.7+ mitmproxy OpenAI GPT Base64 / Hex / JWT / Hash
Instalación rápida Documentación GitHub

Características principales

Interceptación total

HTTP/HTTPS, WebSockets, headers, tokens, parámetros y cuerpos de petición/respuesta.

Encoder / Decoder

Base64, Hex, URL, HTML, ROT13, JWT, MD5, SHA1, SHA256, SHA512. Detección automática de formato.

IA con OpenAI

Análisis automático de tráfico, detección de vulnerabilidades y sugerencias de seguridad (GPT‑3.5/4).

Almacenamiento en .txt

Cada intercambio se guarda en archivos con nombre host_método_numero.txt. Extrae tokens JWT, UUIDs e IDs automáticamente.

Instalación y arranque

1
Clona el repositorio o descarga Auditron.py
git clone https://github.com/Pavlov666/Auditron.git o descarga el archivo directo.
2
Crea un entorno virtual (recomendado)
python3 -m venv auditron-env
source auditron-env/bin/activate
3
Instala mitmproxy y dependencias (IA opcional)
pip install mitmproxy openai (para IA) o solo pip install mitmproxy.
4
Ejecuta mitmproxy con Auditron
mitmproxy -s Auditron.py (interfaz interactiva)
mitmdump -s Auditron.py (solo logs).

Los logs se guardan en ~/auditron_captures/ por defecto. Puedes cambiarlo con export AUDITRON_STORAGE_DIR="/tu/ruta".

Configuración del navegador y certificado CA

Para interceptar tráfico HTTPS, debes instalar el certificado de mitmproxy y configurar el proxy.

1. Inicia mitmproxy con Auditron.
2. Abre el navegador y visita http://mitm.it
3. Descarga el certificado para Linux y añádelo a las autoridades de confianza.
4. Configura el proxy manual en el navegador: HTTP/HTTPS → 127.0.0.1:8080

Firefox: Ajustes → Configuración de red → Proxy manual.
Chromium: lanzar con --proxy-server="http://127.0.0.1:8080" o usar extensión FoxyProxy.

Uso interactivo (mitmproxy)

/

Seleccionar flujo

e / E

Editar request / response (inyectar valores, cambiar headers, body)

r

Replay (repetir solicitud)

f

Filtrar flujos: ~m POST, ~d example.com, ~b secret

S

Guardar flujo seleccionado en archivo .mitm

:script

Ejecutar código Python personalizado

Extracción automática

Tokens JWT, UUIDs, IDs en parámetros con "id". Se guardan en el archivo .txt generado.

Inyección en tiempo real

Modifica peticiones antes de que lleguen al servidor.

Ejemplo de inyección de token:
1. Selecciona el flujo con la cabecera Authorization.
2. Pulsa e → request → busca la línea "Authorization: Bearer ..."
3. Cambia el token, guarda (Ctrl+O) y sal (Ctrl+X)
4. Pulsa a para enviar la petición modificada.

Codificación/Decodificación desde terminal

Auditron puede usarse como herramienta independiente para transformar datos.

# Identificar codificación
python3 Auditron.py --identify --data "SGVsbG8gV29ybGQ="

# Codificar a Base64
python3 Auditron.py --encode base64 --data "Hola mundo"

# Decodificar JWT
python3 Auditron.py --decode jwt --data "eyJhbGciOiJIUzI1NiIs..."

# Generar hash SHA256
python3 Auditron.py --encode sha256 --data "texto sensible"

Formatos soportados: base64, hex, url, html, rot13, jwt, md5, sha1, sha256, sha512.

Puedes agregar codificadores personalizados mediante register_custom_encoder("nombre", función.__code__) desde la consola de mitmproxy o editando el script.

Integración con OpenAI (GPT-3.5/4)

Auditron puede analizar automáticamente cada flujo interceptado con IA o bajo demanda. Sigue estos pasos:

1
Obtén tu API Key de OpenAI en https://platform.openai.com/api-keys.
2
Exporta la clave antes de ejecutar:
export OPENAI_API_KEY="sk-..." (puedes añadirla a ~/.bashrc).
3
Instala la dependencia en tu entorno virtual:
pip install openai
4
Ejecuta mitmproxy con el script modificado (incluye la función analyze_with_ai con OpenAI). El script ya está preparado, solo activa la variable de entorno.

Por defecto el análisis se ejecuta después de cada respuesta. Puedes modificarlo en response() para analizar solo dominios específicos o bajo demanda con :script exec analyze_current_flow().

# Uso interactivo desde la línea de comandos de mitmproxy
:script exec analyze_current_flow()

Nota: Cada análisis consume tokens de OpenAI. Usa con moderación o activa solo para flujos específicos.

Instalación automática (auditron-setup.sh)

El repositorio incluye un script que crea el entorno virtual, instala dependencias y configura la IA de forma interactiva.

chmod +x auditron-setup.sh
./auditron-setup.sh

Te preguntará por tu API key de OpenAI (opcional), descargará el código y lanzará mitmproxy con Auditron.

Resolución de problemas comunes

ProblemaSolución
El navegador no carga páginasVerifica que mitmproxy esté ejecutándose y el proxy configurado (127.0.0.1:8080). Instala el certificado CA en el navegador.
No se crean archivos .txtComprueba permisos de escritura en la carpeta de destino. Ejecuta con -v para ver logs.
No se extraen tokens JWTRevisa que el token siga el patrón eyJ.... Puedes modificar la función extract_tokens en Auditron.py.
La IA no respondeAsegúrate de que OPENAI_API_KEY esté definida, que la librería openai esté instalada y que la cuenta tenga créditos.
